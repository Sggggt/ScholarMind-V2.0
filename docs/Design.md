## 一、系统概述

ScholarMind 是一个端到端的 AI 驱动自动化科研系统，旨在将研究人员从繁琐的重复性工作中解放出来。系统通过串联 9 个精心设计的模块，实现从研究主题输入到学术论文输出的全流程自动化。

### 1.1 核心目标

- **降低科研门槛**: 研究者只需输入研究主题，系统自动完成文献调研、Idea 生成、实验设计、代码生成、实验执行、结果分析、论文写作和同行评审
- **提高研究效率**: 全流程自动化，大幅缩短研究周期
- **保证研究质量**: 多阶段质量控制，自动回退机制确保实验结果达标

### 1.2 系统特色

| 特色 | 说明 |
|------|------|
| **全流程自动化** | 9 个模块无缝串联，无需人工干预 |
| **多 LLM 兼容** | 支持智谱、DeepSeek、OpenAI、本地模型 |
| **智能回退机制** | 实验不达标自动重新执行 |
| **高质量论文** | 5 阶段精细化写作，LaTeX 格式 |
| **专业评审** | NeurIPS 风格多审稿人模拟 |
| **三端支持** | 后端服务 + 桌面 Web + 移动 App |

### 2.3 通信协议

#### REST API

```
基础路径: /api
主要端点:
  - 任务管理: /tasks, /tasks/{id}, /tasks/{id}/pause|resume|abort
  - 任务数据: /tasks/{id}/logs, /tasks/{id}/output, /tasks/{id}/ideas
  - 聊天会话: /chat/sessions, /chat/sessions/{id}/messages
  - 配置: /settings/runtime, /connection-info
  - 健康检查: /health
  - 运行时配置(v2): /runtime-settings GET/PUT
```

#### WebSocket

```
全局连接: ws://host/api/ws?client_type=desktop|mobile
任务连接: ws://host/api/ws/{task_id}?client_type=desktop|mobile

消息类型:
  - progress: 进度更新
  - result: 模块结果
  - need_review: 需要人工审阅
  - error: 错误通知
  - completed: 任务完成
  - ping/pong: 心跳保持
  - agent_tree(v2): 代理树状态整体更新
  - agent_event(v2): 单条代理事件推送
  - agent_summary(v2): 周期级代理摘要
```

#### 局域网发现（mDNS / Bonjour）

为了降低移动端首次接入门槛，系统在固定地址配置之外增加了一条局域网自动发现链路。

设计目标：

- 手机与后端位于同一 Wi-Fi 时，尽量不要求用户手动输入 IP
- 自动发现只负责“找到候选服务”，最终仍以 HTTP / WebSocket 校验为准
- 自动发现失败时，始终可以退回到手动地址模式

发现链路：

```text
backend/ 发布 _scholarmind._tcp.local.
  -> mobile/ 原生应用通过 react-native-zeroconf 扫描
  -> 校验 GET /api/health
  -> 探测 /api/ws
  -> 保存发现到的后端地址
```

工程约束：

- 该能力主要面向移动端原生 App
- AVD 模拟器中的 mDNS 多播不稳定，因此更适合用真机验证
- 桌面端当前不做 mDNS 扫描，但复用同一套 `/api/connection-info`

## 4.5 多代理运行时架构（v2 新增）

M6 阶段采用多代理树架构，coordinator 负责调度，worker 负责具体执行：

```
┌─────────────────────────────────────────────────────┐
│              AgentRuntime (agent_runtime.py)          │
│  维护: active_cycle, root_agent, child_agents,       │
│        recent_summary                                 │
│  广播: agent_tree / agent_event / agent_summary       │
│  通过 WebSocket 实时推送到前端                         │
└────────────────────────┬────────────────────────────┘
                         │
              ┌──────────▼──────────┐
              │    Coordinator      │
              │  (根代理)           │
              │  - 调度 worker      │
              │  - 决策回卷/重试     │
              │  - 压缩反馈         │
              └──────────┬──────────┘
                         │
           ┌─────────────┼─────────────┐
           ▼             ▼             ▼
    ┌────────────┐ ┌────────────┐ ┌────────────┐
    │code-worker │ │env-worker  │ │exp-worker  │
    │代码修改     │ │环境管理     │ │实验执行     │
    │(Aider)     │ │(.venv)     │ │(subprocess)│
    └────────────┘ └────────────┘ └────────────┘
```

**关键特性**：
- **本地实验环境**：`local_env_setup.py` 在项目目录下创建 `.venv-experiment`，通过 SHA256 指纹校验避免重复构建
- **实验循环**：支持多轮实验执行（run_1, run_2, ...），自动重试失败实验
- **压缩反馈**：实验失败时压缩错误信息反馈给 coordinator
- **回卷机制**：coordinator 可决定回卷到 M4 重新规划代码

## 4.6 子进程容错机制（v2 新增）

### 进程组杀灭

所有子进程使用 `start_new_session=True` 创建独立进程组，超时/取消时用 `os.killpg(pgid, SIGKILL)` 杀灭整棵进程树，避免子进程变为僵尸。

### 超时 deadline 模式

`TaskStateMachine.run_interruptible()` 新增 `deadline` 参数（单调时钟绝对时间），修复了原 `asyncio.shield()` 轮询模式导致总超时永远不触发的问题。

### Aider Tags Cache 优化

Docker 环境下将 Aider 的 `.aider.tags.cache.v4/` 重定向到容器本地 `/tmp/` 路径，避免 9P volume I/O 阻塞。

### 僵尸 Git 进程清理

Aider preflight 阶段扫描并清理属于当前仓库路径的僵尸 git 进程。

## 五、数据模型设计

### 5.1 核心数据表

**Task (任务表)**:
```python
- id: 任务唯一标识
- title: 任务标题
- topic: 研究主题描述
- domain: 研究领域
- status: 任务状态 (pending/running/paused/review/completed/failed/aborted)
- current_module: 当前执行模块 (0-9)
- current_step: 当前执行步骤
- progress: 进度百分比 (0-100)
- retry_count: 重试次数
- config: 任务级自定义配置 (JSON)
- created_at/updated_at: 时间戳
```

**TraceLog (日志表)**:
```python
- id: 日志 ID
- task_id: 关联任务
- module: 模块编号 (1-9)
- module_name: 模块名称
- step: 执行步骤
- level: 日志级别 (info/warn/error/debug)
- message: 日志消息
- input_data/output_data: 输入输出数据 (JSON)
- token_usage: Token 使用量
- duration_ms: 执行时长
- timestamp: 时间戳
```

**TaskOutput (产出物表)**:
```python
- id: 产出物 ID
- task_id: 关联任务
- module: 来源模块
- output_type: 产物类型
- file_path: 文件路径
- content: 短内容直接存储
- extra_data: 额外元数据 (JSON)
- created_at: 创建时间
```
