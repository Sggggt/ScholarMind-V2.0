# ScholarMind 前端任务控制增强 — 修改计划书

> **目标**：为 react-client 前端增加流程回退/重启、AI 主动反馈、任务过场动画，实现  
> 「对话推进任务 → 任务反馈 AI → AI 建议用户 → 用户操控任务」的闭环架构。

---

## 一、现状分析

### 1.1 已有能力

| 能力 | 位置 | 说明 |
|------|------|------|
| 暂停/恢复/终止任务 | `AgentRunPage.tsx` | 仅在 Agent 运行页有按钮，主聊天页无入口 |
| 任务完成消息 | `taskAdapter.ts → buildTaskCompletedMessage` | 固定一句话 + quickActions 导航 |
| WebSocket 实时推送 | `WorkspaceSync.tsx` | 支持 `progress / result / need_review / error / completed` |
| 聊天对话 | `MainChatWorkspacePage.tsx` | 用户发消息 → 后端响应 → 显示对话 |
| 任务状态 ribbon | `MainChatWorkspacePage.tsx` | 静态展示当前任务/状态/进度 |

### 1.2 缺失能力

1. **任务重置/回退** — 用户无法重启任务或回退到某阶段重跑
2. **AI 主动引导** — 每个模块完成后 AI 不会主动询问用户下一步
3. **过场动画** — 任务创建/模块切换/完成时无动画反馈
4. **主聊天页任务操控** — 暂停/恢复/终止/重启只在 AgentRunPage，用户需跳页才能操作
5. **quickActions 仅支持导航** — 不支持"点按钮触发 store action"（如暂停/重启）

---

## 二、目标架构

```
用户发消息 ──► 对话推进任务 ──► AI 通过 WebSocket 监控任务 ──► 模块完成时 AI 主动回复
     ▲                                                              │
     │                                                              ▼
     └─────── 用户根据 AI 建议操作（重置/跳过/继续/回退）◄──── AI 建议用户下一步
```

**核心理念**：用户始终在主聊天页操作，AI 是任务的"项目经理"，每个阶段结束后主动汇报并给出选项。

### 3.1 类型层 — `types/app.ts`

**新增类型**：

```ts
// 任务操控指令
export type TaskCommand = 'pause' | 'resume' | 'abort' | 'restart';

// 过场动画状态
export type TransitionState = 'creating' | 'stage-change' | 'completed' | 'restarting' | 'aborting';
```

**修改接口**：

```ts
// ChatQuickAction 增加 command 字段（与 path 二选一）
export interface ChatQuickAction {

  path?: string;        // 导航类：点击跳转
  stageId?: StageId;
  command?: TaskCommand; // 命令类：点击触发 store action
}

// ChatMessage.kind 增加 'stage-transition' 类型
export interface ChatMessage {
  // ...
  kind?: 'text' | 'thinking' | 'stage-transition';
}
```

### 3.3 适配器层 — `adapters/taskAdapter.ts`

**新增函数**：

```ts
// 1. 构造模块完成时的 AI 主动反馈消息
// 例：M1 完成 → "✅ 文献研究阶段已完成 / 已检索核心文献 / 接下来你可以：[查看结果] [进入下一步] [暂停] [终止]"
export function buildModuleCompletedMessage(
  task: BackendTaskResponse,
  moduleId: string,      // e.g. 'M1'

// 2. 构造阶段切换提示消息（kind='stage-transition'，带动画渲染）
export function buildStageTransitionMessage(
  task: BackendTaskResponse,
  newModuleId: string,
): ChatMessage

// 3. 根据当前任务状态生成可用的任务操控按钮
export function buildTaskControlActions(runStatus: RunStatus): ChatQuickAction[]
```

**修改函数**：

```ts
// buildTaskCompletedMessage 增强 → 丰富完成语 + 增加"重启任务"按钮
export function buildTaskCompletedMessage(task: BackendTaskResponse): ChatMessage
```

**新增数据映射**：

```ts
// 模块 → 阶段信息映射（用于生成消息和导航）
const moduleToStageMap: Record<string, { stageId, path, title }>
// 模块 → 下一模块映射（用于"进入下一步"按钮）
```

### 3.4 Store 核心 — `store/useWorkspaceStore.ts`

**新增状态字段**：

```ts
// 过场动画控制
transitionState: TransitionState | null;
transitionLabel: string;

// 防止重复注入模块完成消息
lastCompletedModule: string;
```

**新增 actions**：

```ts
// 重启任务（abort 当前 → 用相同 topic 创建新任务）
restartCurrentTask: () => Promise<void>;

// 过场动画控制
showTransition: (state: TransitionState, label: string) => void;
clearTransition: () => void;

// 执行 quickAction 中的任务命令
executeTaskCommand: (command: TaskCommand) => Promise<void>;
```

**增强 `handleWsMessage`**：

```ts
// 当收到 progress 消息 & module 字段变化时：
// 1. 对比 lastCompletedModule，若不同 → 旧模块已完成
// 2. 调用 buildModuleCompletedMessage 生成 AI 反馈消息
// 3. 调用 buildStageTransitionMessage 生成阶段切换消息
// 4. 注入到当前 session 的聊天流
// 5. 触发 showTransition('stage-change', '正在进入 xxx 阶段...')

// 当收到 completed 消息时：
// 1. 触发 showTransition('completed', '研究任务已完成')
```

**增强 `addChatMessage`**：

```ts
// 当后端响应 task_created === true 时：
// 触发 showTransition('creating', '正在初始化研究任务...')
```

**增强 `restartCurrentTask`**：

```ts
// 1. 触发 showTransition('restarting', '正在重启任务...')
// 2. 调用 restartTask API（fallback: abortTask + createTask）
// 3. 更新 session 绑定的 task
// 4. 注入 AI 消息："任务已重启，正在重新从文献研究开始..."
```

### 3.5 新组件 — `components/ui/TaskCommandBar.tsx`

**功能**：浮在聊天区底部输入框上方的任务控制条。

```
┌──────────────────────────────────────────────────────────┐
│  📋 联邦学习跨域泛化  │  M3 构思评分  │  42%  │  ⏸ 暂停  │  ⏹ 终止  │  🔄 重启  │
└──────────────────────────────────────────────────────────┘
```

- 任务标题 + 当前模块 + 进度百分比
- 操作按钮根据 `runStatus` 动态显示/隐藏
- 无任务时隐藏（`currentTask === null` 时不渲染）
- 紧凑布局，不占太多空间

---

### 3.6 新组件 — `components/ui/TaskTransitionOverlay.tsx`

**功能**：非阻塞的半透明过场动画浮层。

- 任务创建时：「正在初始化研究任务...」+ 脉冲圆环动画
- 模块切换时：「正在进入趋势分析阶段...」+ 滑入动画
- 任务完成时：「研究任务已完成 ✓」+ 淡出动画
- 2–3 秒后自动消失，`pointer-events: none` 不阻塞交互
- 定位在 `AppLayout` 层级，覆盖在内容区上方

## 四、后端依赖说明

| 端点 | 当前状态 | 前端处理方式 |
|------|---------|-------------|
| `POST /tasks/{id}/pause` | ✅ 已存在 | 直接调用 |
| `POST /tasks/{id}/resume` | ✅ 已存在 | 直接调用 |
| `POST /tasks/{id}/abort` | ✅ 已存在 | 直接调用 |
| `POST /tasks/{id}/restart` | ❌ 不存在 | **fallback**：abort → 用相同 topic 调用 createTask |
| `POST /tasks/{id}/reset-module` | ❌ 不存在 | **预留**：调用时 toast 提示 "该功能需要后端支持" |
| WebSocket `module_completed` 事件 | ❌ 不存在 | **兼容方案**：对比 progress.module 字段变化 + percent=100 推断 |
