# ScholarMind 自动化科研系统 — 现有开源项目调研与系统设计

## 一、系统概述

ScholarMind 是一个端到端的 AI 驱动自动化科研系统，旨在将研究人员从繁琐的重复性工作中解放出来。系统通过串联 9 个精心设计的模块，实现从研究主题输入到学术论文输出的全流程自动化。

### 1.1 核心目标

- **降低科研门槛**: 研究者只需输入研究主题，系统自动完成文献调研、Idea 生成、实验设计、代码生成、实验执行、结果分析、论文写作和同行评审
- **提高研究效率**: 全流程自动化，大幅缩短研究周期
- **保证研究质量**: 多阶段质量控制，自动回退机制确保实验结果达标

### 1.2 系统特色

| 特色 | 说明 |
|------|------|
| **全流程自动化** | 9 个模块无缝串联，无需人工干预 |
| **多 LLM 兼容** | 支持智谱、DeepSeek、OpenAI、本地模型 |
| **智能回退机制** | 实验不达标自动重新执行 |
| **高质量论文** | 5 阶段精细化写作，LaTeX 格式 |
| **专业评审** | NeurIPS 风格多审稿人模拟 |

## 四、基础设施层技术实现

### 4.1 LLM 客户端 (modules/llm_client.py)

**功能**:
- 统一 OpenAI 兼容接口
- 自动重试机制 (指数退避 + 抖动)
- 超时控制 (连接 30s, 读取 180s)
- 支持 JSON 模式

**接口**:
```python
async def call_llm(
    prompt: str,
    system: str = "You are a professional research assistant.",
    model: str | None = None,
    temperature: float = 0.7,
    max_tokens: int = 4096,
) -> tuple[str, int]:  # (response_text, token_usage)

async def call_llm_json(
    prompt: str,
    system: str = "You are a professional research assistant. Reply in JSON.",
) -> tuple[dict, int]:  # (parsed_json, token_usage)
```

**支持的服务商**:
- 智谱 AI (推荐国内用户)
- DeepSeek
- OpenAI
- 本地模型 (Ollama, vLLM, LM Studio)

### 4.2 搜索服务集成 (modules/ai_scientist_bridge.py)

```python
- topic: 研究主题描述
- domain: 研究领域
- status: 任务状态 (pending/running/paused/review/completed/failed/aborted)
- current_module: 当前执行模块 (0-9)
- current_step: 当前执行步骤
- progress: 进度百分比 (0-100)
- retry_count: 重试次数
- config: 任务级自定义配置 (JSON)
- created_at/updated_at: 时间戳
```

**TraceLog (日志表)**:
```python
- id: 日志 ID
- task_id: 关联任务
- module: 模块编号 (1-9)
- module_name: 模块名称
- step: 执行步骤
- level: 日志级别 (info/warn/error/debug)
- message: 日志消息
- input_data/output_data: 输入输出数据 (JSON)
- token_usage: Token 使用量
- duration_ms: 执行时长
- timestamp: 时间戳
```

**TaskOutput (产出物表)**:
```python
- id: 产出物 ID
- task_id: 关联任务
- module: 来源模块
- output_type: 产物类型
- file_path: 文件路径
- content: 短内容直接存储
- extra_data: 额外元数据 (JSON)
- created_at: 创建时间
```

**ChatSession (会话表)**:
```python
- id: 会话 ID
- title: 会话标题
- summary: 会话摘要
- task_id: 关联任务 (可选)
- created_at/updated_at: 时间戳
```

**ChatMessage (消息表)**:
```python
- id: 消息 ID
- session_id: 关联会话
- role: 角色 (user/assistant)
- kind: 消息类型 (text/thinking/running-status)
- content: 消息内容
- extra_data: 额外数据 (JSON)
```
